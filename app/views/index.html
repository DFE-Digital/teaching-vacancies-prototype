Notes on prototyping

  • Where possible prototype full flows and join them up to existing flows.
  • Avoid isolated, static screens. If you do need to create them, include links to the list on this screen so it's easy for everyone to find them.
  • If you do need to create a dead end, add a hash to the anchor to alert the user. Example: This link will notify the user that they can't go any further
  • Avoid versioning your prototypes, this leads to unnecessary duplicition of code. You'll never go back to an earlier version, trust me.
